Спасибо, что помогаете делать CyberDeck безопаснее.

## Поддерживаемые версии

Исправления безопасности применяются по best-effort для:

| Ветка/версия | Статус |
|---|---|
| Последний релиз | Поддерживается |
| `main` | Поддерживается |
| Старые релизы | Без гарантий |

## Как сообщить об уязвимости

Пожалуйста, не публикуйте уязвимость сразу в открытом доступе.

Предпочтительные каналы:

1. Приватный security report на GitHub:  
   <https://github.com/Overl1te/CyberDeck/security/advisories/new>
2. Если недоступно, свяжитесь с мейнтейнером:  
   <https://github.com/Overl1te>

## Что добавить в отчет

- краткое описание проблемы;
- шаги воспроизведения;
- потенциальное влияние;
- затронутые версии/коммиты;
- предполагаемая мера снижения риска (если есть).

## SLA обработки (best effort)

| Этап | Ориентир |
|---|---|
| Подтверждение получения | до 72 часов |
| Первичный triage | до 7 дней |
| План фикса/mitigation | до 14 дней |

Для сложных случаев сроки могут увеличиваться.

## Раскрытие информации

- Сначала согласуется исправление, затем публичное раскрытие.
- По желанию репортера указывается авторство (credit).
- После релиза фикса детали могут быть опубликованы для прозрачности.
